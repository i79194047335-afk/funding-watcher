## Что это за проект

Бот-наблюдатель за ставками фандинга (funding rate) на бирже Hyperliquid.
Смотрит перпы (бессрочные контракты) по всем монетам и подсвечивает те, где
перекос ставки большой — это кандидаты на market-neutral стратегию
(спот + противоположный шорт/лонг, чтобы собирать фандинг независимо от цены).

**Важно про деньги:** реальная цель не доход (на бюджете ~$200 это центы),
а навык + рабочий бот для портфолио/фриланса. На старте бот ничем не рискует —
он только смотрит и считает. Реальные сделки — позже и крошечные.

## Стратегия в одну фразу

Фандинг капает каждый час. `+` ставка → лонги платят шортам (собираем, будучи в шорте).
`−` → шорты платят лонгам (собираем в лонге). Бот помечает кандидата от
`MIN_FUNDING_PCT = 0.05`%/ч (порог «обратить внимание»); реальная сделка отбивает
комиссии примерно от **~0.1%/ч** — это выше, цифры подгоняем по наблюдению.
На мейджорах край сжат, живёт длинный хвост (мелкие/свежие монеты). Гонка ботов
(MEV/Solana/мосты/стейблы) — не наш профиль, бюджет и латентность не те.

Первую реальную сделку делаем только на сигнале со знаком `+` (хедж = купить спот +
шорт перп). На `−` нужно шортить спот — новичку на HL недоступно, такие пропускаем.

## Технические детали

- Python 3.10, библиотеки `requests`, `python-dotenv` (есть в системе; также `pandas`/`numpy`).
- API Hyperliquid: `POST https://api.hyperliquid.xyz/info`, без ключей и регистрации.
  - Перпы: `{"type": "metaAndAssetCtxs"}` → `data[0]["universe"]` (имена),
    `data[1]` (контексты: `funding` — ставка/час в долях, `dayNtlVlm` — объём 24ч в $).
  - Спот: `{"type": "spotMetaAndAssetCtxs"}`. Нативный спот против USDC есть лишь
    у ~8 из 230 перпов — это фильтр «есть чем хеджить».
- Файлы: `watcher.py` (логика: запрос, фильтр, уведомления, история), 
  `telegram_notify.py` (отправка в Telegram), `get_chat_id.py`
  (помощник найти chat_id), `STEP6_GUIDE.md` (памятка по первой сделке).
- Пороги в начале `watcher.py`: `MIN_FUNDING_PCT=0.05` (наблюдаем),
  `MIN_ACTIONABLE_PCT=0.10` (отбивает комиссии, сигнал в Telegram),
  `MIN_VOLUME_USD=1_000_000`.
- Кандидаты делятся на три группы: ✅ `actionable` (≥0.10%), 👀 `watchlist` (0.05–0.10%),
  ✗ `skip_negative` (≤0, недоступен без шорта спота). В Telegram — только ✅.
- Антиспам + сигнал на выход: отслеживаем actionable в `data/notified_state.json`,
  при появлении → `notify_candidates()`, при исчезновении → `notify_exited()`.
- История в `data/history.csv`: timestamp, coin, hourly_pct, annual_pct, volume_usd, has_spot.
  Пишется каждый запуск, все 230 монет. Растёт ~22K строк/день (~50 МБ/месяц).
- Логирование: `log()` пишет в `logs/watcher.log` (ошибки API, входы/выходы кандидатов).
- API-запросы обёрнуты в try/except: при сбое пишем в лог, не падаем.
  Проверяются ОБА ответа (spot_bases и rates) — частичный сбой не маскируется.
- Запуск разово: `python3 watcher.py` из `/root/projects/funding-watcher`.
  24/7: cron `*/15 * * * * cd <проект> && /usr/bin/python3 watcher.py >> logs/watcher.log 2>&1`.
- Папки `logs/` и `data/` существуют и используются, но в git не попадают.
