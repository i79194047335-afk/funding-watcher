Сначала пара слов простыми словами
Арбитраж — это «купил дешевле тут, продал дороже там, разницу забрал». Одна и та же вещь стоит чуть по-разному в двух местах — ты на этом зарабатываешь.

DEX — биржа без хозяина, обмен крипты идёт сам собой через программу (смарт-контракт). CEX — обычная биржа с компанией внутри (Binance и т.п.).

Ликвидность — сколько денег «налито» в конкретный рынок. Много денег = можно купить/продать, не двигая цену. Мало = любое движение дёргает цену.

Газ — комиссия сети за каждую операцию. Слиппедж (проскальзывание) — пока сделка идёт, цена чуть уехала, и ты получил хуже, чем хотел.

Про что был весь чат, одной мыслью
Арбитраж бывает на крупных монетах (биткоин, эфир — назовём их «большие») и на мелких, малоизвестных («длинный хвост»).

На больших монетах разницу в цене мгновенно сжирают роботы-профессионалы. Это уже не «нашёл и забрал», а спорт за миллисекунды: у кого сервер физически стоит в том же здании, что и сервер сети, тот и выиграл. Туда новичку с маленьким бюджетом — никак. Это главный вывод. Поэтому всё интересное — в мелочи, где разницы живут дольше и крупным игрокам неинтересно из-за маленьких сумм.

«Вечный контракт» (перп) — это ставка на цену монеты без срока. Чтобы цена этой ставки не убегала от настоящей цены монеты, каждый час одна сторона доплачивает другой. Вот эта доплата и есть фандинг.

Фокус такой: ты одновременно покупаешь монету по-настоящему и ставишь «вниз» на том же объёме.

Цена пошла вверх → твоя монета подорожала, ставка «вниз» в минусе → итог ноль.
Цена пошла вниз → наоборот → снова ноль.
Тебе всё равно, куда идёт цена. А каждый час ты просто собираешь фандинг. На мелких монетах он иногда жирный. Денег ты не угадываешь — в этом вся прелесть, это не «казино».

Шаг 2 из 6 — Первый запрос к Hyperliquid

Hyperliquid отдаёт данные бесплатно, без регистрации и ключей. Просто спрашиваем — он отвечает.

Открой файл watcher.py и вставь этот код:

python
import requests
import json
from datetime import datetime

# Адрес API Hyperliquid — бесплатно, без ключей
URL = "https://api.hyperliquid.xyz/info"

def get_funding_rates():
    """Запрашиваем текущие ставки фандинга по всем монетам"""
    payload = {"type": "metaAndAssetCtxs"}
    
    response = requests.post(URL, json=payload)
    data = response.json()
    
    # data[0] — список монет с названиями
    # data[1] — список данных по каждой монете
    coins = data[0]["universe"]
    contexts = data[1]
    
    results = []
    
    for i, coin in enumerate(coins):
        name = coin["name"]
        ctx = contexts[i]
        
        # fundingRate — текущая ставка за час (в долях, не в процентах)
        funding_rate = float(ctx["funding"])
        
        # Переводим в проценты и считаем годовых
        hourly_pct = funding_rate * 100        # % в час
        annual_pct = hourly_pct * 24 * 365    # % годовых (грубо)
        
        results.append({
            "coin": name,
            "hourly_%": round(hourly_pct, 4),
            "annual_%": round(annual_pct, 1)
        })
    
    return results

def main():
    print(f"\n[{datetime.now().strftime('%H:%M:%S')}] Запрашиваем данные...\n")
    
    rates = get_funding_rates()
    
    # Сортируем по модулю — самые большие отклонения наверху
    rates.sort(key=lambda x: abs(x["hourly_%"]), reverse=True)
    
    print(f"{'Монета':<12} {'% в час':>10} {'% годовых':>12}")
    print("-" * 36)
    
    # Показываем топ-20
    for r in rates[:20]:
        hourly = r["hourly_%"]
        annual = r["annual_%"]
        
        # Помечаем интересные (больше 0.1% в час)
        flag = " ★" if abs(hourly) > 0.1 else ""
        
        print(f"{r['coin']:<12} {hourly:>10.4f} {annual:>11.1f}%{flag}")
    
    print(f"\nВсего монет: {len(rates)}")
    print("★ = больше 0.1% в час (интересно для стратегии)")

if __name__ == "__main__":
    main()

Что значат эти цифры
Знак (плюс/минус) — кто кому платит:

Плюс (HEMI +0.0392) — те, кто ставит «вверх» (лонги), платят тем, кто «вниз» (шортам). Значит на рынке перекос в сторону лонгов, толпа набилась в покупку. Чтобы собирать этот фандинг → нужно быть в шорте (плюс держать спот для нейтральности).
Минус (BRETT −0.0183, TON −0.0110) — наоборот, шорты платят лонгам. Толпа набилась в шорт/панику. Чтобы собирать → быть в лонге.
«% годовых» — это если бы ставка держалась такой весь год. Она так не держится, цифра прыгает каждый час. Это просто чтобы почувствовать масштаб. HEMI «343% годовых» в моменте — это не «заработаешь 343%», это «прямо сейчас перекос очень сильный».

Что важное видно
Ни одной монеты с ★ кроме HEMI — то есть прямо сейчас рынок довольно спокойный, больших перекосов мало. И это нормально. Жирные ставки появляются волнами — на новостях, на свежих листингах, в панику. Наблюдатель и нужен, чтобы ловить эти моменты, а не пялиться в экран.

HEMI +0.0392%/час — единственный кандидат. Помнишь порог из нашего разбора? Сделка имеет смысл примерно от 0.1%/час (чтобы комиссии отбить). HEMI пока ниже, но близко — как раз такие и надо ловить, когда дорастут.

# 2026-06-17 — Ревью и доработки (DeepSeek + разработчик)

## Что произошло

Провели двухраундовое ревью кода funding-watcher. Первый раунд: DeepSeek
проанализировал код по описанию (без доступа к исходникам) и дал 8 пунктов.
Второй раунд: разработчик сверил ревью с живым кодом — все 8 подтверждены,
добавлено 4 новых находки (видно только при чтении кода).

По итогам реализовано 6 правок (3 коммита):

### Правка 1 — Сигнал на выход (коммит 16d62c2)
- Бот молчал, когда кандидат исчезал. Теперь `notify_exited()` шлёт в Telegram:
  «XYZ: ставка вернулась к норме → пора закрывать позицию».
- Цикл сделки «вошёл → слежу → вышел» замкнут.

### Правка 2 — Защита от минусовых сигналов (коммит 16d62c2)
- Раньше для отрицательного фандинга бот писал «→ лонг перп + спот» — это
  двойной лонг, а не нейтраль (активно неверная инструкция).
- Теперь в Telegram уходят только положительные (+) сигналы. Отрицательные
  в терминале помечены «✗ недоступен (нужен шорт спота)».

### Правка 3 — Два порога (коммит 2215b3e)
- `MIN_ACTIONABLE_PCT = 0.10` — порог «отбивает комиссии» (зелёный).
  `MIN_FUNDING_PCT = 0.05` — порог «наблюдаем» (жёлтый).
- Три группы в терминале: ✅ доступны / 👀 наблюдаем / ✗ пропускаем.
- Telegram: только ≥0.10%.
- Пример в STEP6_GUIDE исправлен с +0.0850% на +0.1200%.

### Правка 4 — try/except + логирование (коммит 2215b3e)
- Оба API-запроса обёрнуты в try/except: при ошибке пишем в лог, не падаем.
- `log()` пишет в `logs/watcher.log` с меткой времени (папка `logs/` теперь
  реально используется).
- События (новый кандидат, выход) тоже логируются.

### Правка 5 — История в CSV (коммит 2215b3e)
- `data/history.csv`: timestamp, coin, hourly_pct, annual_pct, volume_usd, has_spot.
- 230 строк за запуск, ~22K/день. Позволяет видеть тренд: всплеск или устойчиво.

### Правка 6 — Частичный сбой API (коммит 5ec25b0)
- Если get_spot_bases() падает, а get_funding_rates() — нет, бот раньше врал
  «кандидатов нет» (has_spot=False у всех → кандидатов 0).
- Теперь проверяются оба ответа. + save_history() обёрнут в try/except.
